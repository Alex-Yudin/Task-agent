# Android-приложение «Орбита»

Версия `0.3.0` — нативное APK-приложение с локальной SQLite-базой и синхронизацией через Google Таблицу или Windows.

## Возможности

- работа без интернета;
- проекты, задачи, статусы и приоритеты;
- голосовое создание проекта или задачи на русском языке;
- локальное хранение на телефоне;
- двусторонняя синхронизация через Google Таблицу из любой сети;
- резервная локальная синхронизация с Windows в одной Wi‑Fi-сети;
- разрешение конфликтов по времени последнего изменения;
- отдельный случайный токен для доступа к данным;
- автоматическая сборка APK через GitHub Actions.

## Получение APK

1. Откройте репозиторий на GitHub.
2. Перейдите на вкладку **Actions**.
3. Откройте последний запуск **Build Android APK**.
4. В блоке **Artifacts** скачайте `Orbita-Android-v0.3.0`.
5. Распакуйте ZIP и установите APK на телефон.

Android может запросить разрешение на установку приложений из браузера или файлового менеджера. Разрешите установку только для использованного приложения, затем при желании отключите разрешение обратно.

## Рекомендуемый режим: Google Таблица

1. Выполните настройку по [GOOGLE_SHEETS.md](GOOGLE_SHEETS.md).
2. В Android откройте **Настройки**.
3. В поле адреса вставьте URL опубликованного Google Apps Script, оканчивающийся на `/exec`.
4. В поле секрета вставьте значение, показанное функцией `setupOrbita`.
5. Нажмите **Сохранить и синхронизировать**.

После этого телефон синхронизируется через HTTPS и не зависит от запущенного компьютера.

## Резервный режим: Windows в одной Wi-Fi-сети

1. Запустите `start-windows.cmd`. Теперь одновременно работают:
   - Windows-интерфейс: `http://127.0.0.1:3765`;
   - Android-синхронизация: порт `3766`.
2. Нажмите правой кнопкой на `configure-sync-firewall.cmd` и выберите **Запуск от имени администратора**. Скрипт откроет порт 3766 только для частной сети.
3. Дважды запустите `show-sync-info.cmd`. Он покажет:
   - IPv4-адрес компьютера;
   - токен из `data/sync-token.txt`.
4. Убедитесь, что сеть Windows отмечена как **Частная**, а компьютер и телефон подключены к одному Wi‑Fi.

## Настройка Android для локального режима

1. Откройте **Настройки** в нижнем меню приложения.
2. Введите адрес компьютера, например `http://192.168.1.10:3766`.
3. Вставьте токен из `data/sync-token.txt`.
4. Нажмите **Сохранить и синхронизировать**.

Токен нельзя публиковать или передавать посторонним. Если он раскрыт, закройте Windows-приложение, удалите только файл `data/sync-token.txt` и снова запустите приложение — будет создан новый токен.

## Модель синхронизации

Телефон отправляет локальные записи и получает изменения Google Таблицы или Windows. Для записи с одинаковым идентификатором сохраняется вариант с более новым `updatedAt`. Физическое удаление в версии 0.3 не используется: задачи отменяются, проекты архивируются, поэтому удалённая запись не может случайно «воскреснуть».

## Ограничения версии 0.3

- требования запущенного компьютера и общей Wi‑Fi-сети относятся только к локальному режиму;
- локальный HTTP предназначен только для доверенной домашней или рабочей Wi‑Fi-сети;
- APK собран отладочным ключом и предназначен для прямой установки, а не публикации в Google Play.
