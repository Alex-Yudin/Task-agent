# Google OAuth в «Орбите» для Windows

Версия 0.4.2 использует Desktop OAuth Client и напрямую обращается к Google Sheets API. Публичный Apps Script URL и общий секрет синхронизации для Windows больше не требуются.

## Однократная настройка OAuth Client

1. Откройте Google Cloud Console и скачайте JSON для созданного Desktop OAuth Client.
2. В папке «Орбиты» дважды щёлкните `configure-google-oauth.cmd`.
3. В появившемся окне выберите скачанный JSON.
4. Дождитесь сообщения `Google OAuth client configured`.

Мастер проверяет наличие `client_id` и `client_secret`, затем шифрует весь JSON через Windows DPAPI. Client ID из выбранного JSON имеет приоритет над значением по умолчанию в `appsettings.json`, поэтому JSON от перевыпущенного Desktop Client можно подключить без ручного редактирования настроек. Исходный файл из папки загрузок после успешной настройки можно удалить.

## Вход

1. Выполните однократную настройку OAuth Client, описанную выше.
2. Запустите `start-windows.cmd`.
3. В левой панели нажмите **Аккаунт Google**.
4. Нажмите **Войти через Google**.
5. В открывшемся системном браузере выберите аккаунт и разрешите доступ к файлам, создаваемым или открываемым «Орбитой».
6. Вернитесь в приложение: в окне появится адрес подключённого аккаунта.

Авторизация использует Authorization Code с PKCE и callback `http://127.0.0.1:3765/api/google/oauth/callback`. Пароль Google приложение не получает.

## Создание таблицы

После входа нажмите **Создать таблицу «Орбита»**. Приложение создаст листы `Instructions`, `Projects` и `Tasks`, сохранит Spreadsheet ID и запустит первую синхронизацию.

Рекомендуется создавать таблицу именно этой кнопкой. Scope `drive.file` разрешает «Орбите» работать только с файлами, которые пользователь открыл или создал через приложение.

Ранее созданную и уже разрешённую таблицу можно подключить по полной ссылке или Spreadsheet ID.

## Хранение токена

В Windows OAuth Client JSON и refresh token шифруются через DPAPI с областью `CurrentUser`. Зашифрованные данные сохраняются в `data/google-oauth-client.dat` и `data/google-oauth-tokens.dat`; другой пользователь Windows расшифровать их не сможет. Папка `data` исключена из Git.

Client ID находится в `appsettings.json` и не является секретом. `client_secret` никогда не передаётся браузеру: приложение отправляет его только на защищённый token endpoint Google при получении или обновлении токена.

## Совместимость

- Apps Script остаётся резервным режимом;
- при наличии OAuth и выбранной таблицы синхронизация автоматически использует прямой Sheets API;
- Android OAuth добавляется отдельно, поскольку Google требует Android Client ID, связанный с package name и SHA-1 сертификата APK;
- ChatGPT подключается к той же таблице через Google Drive независимо от OAuth-токена «Орбиты».
