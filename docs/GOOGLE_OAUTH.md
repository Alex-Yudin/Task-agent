# Google OAuth в «Орбите» для Windows

Версия 0.4 использует Desktop OAuth Client ID и напрямую обращается к Google Sheets API. Публичный Apps Script URL и общий секрет для Windows больше не требуются.

## Вход

1. Запустите `start-windows.cmd`.
2. В левой панели нажмите **Аккаунт Google**.
3. Нажмите **Войти через Google**.
4. В открывшемся системном браузере выберите аккаунт и разрешите доступ к файлам, создаваемым или открываемым «Орбитой».
5. Вернитесь в приложение: в окне появится адрес подключённого аккаунта.

Авторизация использует Authorization Code с PKCE и callback `http://127.0.0.1:3765/api/google/oauth/callback`. Пароль Google приложение не получает.

## Создание таблицы

После входа нажмите **Создать таблицу «Орбита»**. Приложение создаст листы `Instructions`, `Projects` и `Tasks`, сохранит Spreadsheet ID и запустит первую синхронизацию.

Рекомендуется создавать таблицу именно этой кнопкой. Scope `drive.file` разрешает «Орбите» работать только с файлами, которые пользователь открыл или создал через приложение.

Ранее созданную и уже разрешённую таблицу можно подключить по полной ссылке или Spreadsheet ID.

## Хранение токена

В Windows refresh token шифруется через DPAPI с областью `CurrentUser`. Зашифрованные данные сохраняются в `data/google-oauth-tokens.dat` и не могут быть расшифрованы другим пользователем Windows. Файл исключён из Git.

Client ID находится в `appsettings.json` и не является секретом. Client secret для Desktop-потока не используется.

## Совместимость

- Apps Script остаётся резервным режимом;
- при наличии OAuth и выбранной таблицы синхронизация автоматически использует прямой Sheets API;
- Android OAuth добавляется отдельно, поскольку Google требует Android Client ID, связанный с package name и SHA-1 сертификата APK;
- ChatGPT подключается к той же таблице через Google Drive независимо от OAuth-токена «Орбиты».
