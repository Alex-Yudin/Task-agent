# Орбита — ИИ-менеджер задач

Рабочая Windows- и Android-платформа из спецификации «Интеллектуальная платформа управления задачами через голосовой диалог с ИИ».

## Что уже работает

- проекты, задачи и подзадачи;
- статусы, приоритеты и сроки;
- ежедневный план и контроль просроченных задач;
- текстовые и голосовые команды на русском языке;
- создание проектов и задач из диалога;
- завершение задачи голосовой командой;
- поиск по проектам, задачам, истории и базе знаний;
- история диалогов и журнал всех изменений;
- локальная SQLite-база в безопасном WAL-режиме;
- консистентные резервные копии;
- нативное Android-приложение с офлайн-базой;
- двусторонняя синхронизация Windows и Android;
- единая Google Таблица, доступная «Орбите» и ChatGPT;
- автоматическая выгрузка изменений и загрузка правок из Google Sheets.
- вход через Google OAuth 2.0 в Windows без ручного URL и общего секрета.

## Быстрый запуск в Windows

1. Установите [Node.js 24 LTS](https://nodejs.org/), если он ещё не установлен.
2. Откройте папку проекта.
3. Дважды щёлкните `start-windows.cmd`.
4. Приложение откроется по адресу `http://127.0.0.1:3765`.

Основная локальная копия хранится в `data/tasks.sqlite`. Резервные копии создаются в папке `backups`. После подключения Google Sheets проекты и задачи дополнительно синхронизируются с выбранной пользователем таблицей.

## Google Таблица и ChatGPT

Рекомендуемый способ подключения — Google OAuth по инструкции [docs/GOOGLE_OAUTH.md](docs/GOOGLE_OAUTH.md). Apps Script из [docs/GOOGLE_SHEETS.md](docs/GOOGLE_SHEETS.md) сохранён как совместимый резервный режим. После настройки:

Перед первым входом запустите `configure-google-oauth.cmd` и выберите скачанный из Google Cloud OAuth Client JSON. Локальная копия шифруется Windows DPAPI и не попадает в Git.

- Windows отправляет изменения автоматически и проверяет таблицу каждые пять минут;
- Android может синхронизироваться напрямую через интернет по URL Google Apps Script;
- ChatGPT читает и заполняет ту же таблицу через подключение Google Drive;
- конфликты разрешаются в пользу записи с более новым `updatedAt`.

## Android

APK автоматически собирается во вкладке **Actions → Build Android APK**. Полная инструкция по установке и синхронизации находится в [docs/ANDROID.md](docs/ANDROID.md).

Android поддерживает два режима: Google Таблица через интернет или локальный Windows-сервер в одной Wi‑Fi-сети.

## Примеры команд

```text
Создай проект Судебные дела
Задача: срочно подготовить отзыв до завтра
Заверши задачу подготовить отзыв
Найди Альфа-Банк
План на сегодня
```

Голосовой ввод работает в Microsoft Edge и Google Chrome при наличии разрешения на использование микрофона.

## Команды разработчика

```bash
npm start
npm test
npm run check
```

Сторонние npm-пакеты не требуются. Для работы используется встроенный модуль SQLite из Node.js 24.

## Архитектура

```text
public/                         локальный пользовательский интерфейс
src/server.js                   HTTP/API и composition root
src/application/               сценарии приложения и анализ диалога
src/domain/                     правила и доменная валидация
src/infrastructure/             SQLite, конфигурация и журналирование
test/                           автоматические тесты
```

Зависимости направлены от внешних слоёв к доменным. Модуль анализа диалога реализует локальный детерминированный режим; облачный LLM-провайдер добавляется как отдельный адаптер без изменения задач и хранилища.

## Ограничения итерации 0.4

- распознавание свободной речи пока основано на наборе надёжных команд, без подключения внешней LLM;
- Google Sheets подключён; интеграции почты и календаря ещё не реализованы;
- Windows-приложение запускается локально через Node.js, упаковка в установщик `.exe` относится к следующей итерации;
- для автоматического извлечения задач именно в ChatGPT требуется подключить Google Drive и использовать подготовленные инструкции;
- Android OAuth ожидает отдельный Android Client ID; до его добавления APK использует Apps Script или локальную Windows-синхронизацию;
- шифрование отдельных чувствительных полей и векторная память запланированы после стабилизации основной модели данных.

Подробный статус реализации находится в [docs/IMPLEMENTATION.md](docs/IMPLEMENTATION.md).
