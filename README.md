# Орбита — локальный ИИ-менеджер задач

Первая рабочая итерация Windows-платформы из спецификации «Интеллектуальная Windows-платформа управления задачами через голосовой диалог с ИИ».

## Что уже работает

- проекты, задачи и подзадачи;
- статусы, приоритеты и сроки;
- ежедневный план и контроль просроченных задач;
- текстовые и голосовые команды на русском языке;
- создание проектов и задач из диалога;
- завершение задачи голосовой командой;
- поиск по проектам, задачам, истории и базе знаний;
- история диалогов и журнал всех изменений;
- локальная SQLite-база в безопасном WAL-режиме;
- консистентные резервные копии;
- адаптеры и слои для последующего подключения облачного ИИ и внешних интеграций.

## Быстрый запуск в Windows

1. Установите [Node.js 24 LTS](https://nodejs.org/), если он ещё не установлен.
2. Откройте папку проекта.
3. Дважды щёлкните `start-windows.cmd`.
4. Приложение откроется по адресу `http://127.0.0.1:3765`.

Данные не отправляются в интернет. Они сохраняются в `data/tasks.sqlite`. Резервные копии создаются в папке `backups`.

## Примеры команд

```text
Создай проект Судебные дела
Задача: срочно подготовить отзыв до завтра
Заверши задачу подготовить отзыв
Найди Альфа-Банк
План на сегодня
```

Голосовой ввод работает в Microsoft Edge и Google Chrome при наличии разрешения на использование микрофона.

## Команды разработчика

```bash
npm start
npm test
npm run check
```

Сторонние npm-пакеты не требуются. Для работы используется встроенный модуль SQLite из Node.js 24.

## Архитектура

```text
public/                         локальный пользовательский интерфейс
src/server.js                   HTTP/API и composition root
src/application/               сценарии приложения и анализ диалога
src/domain/                     правила и доменная валидация
src/infrastructure/             SQLite, конфигурация и журналирование
test/                           автоматические тесты
```

Зависимости направлены от внешних слоёв к доменным. Модуль анализа диалога реализует локальный детерминированный режим; облачный LLM-провайдер добавляется как отдельный адаптер без изменения задач и хранилища.

## Ограничения итерации 0.1

- распознавание свободной речи пока основано на наборе надёжных команд, без подключения внешней LLM;
- интеграции Google Drive, почты и календаря подготовлены архитектурно, но не подключены;
- приложение запускается локально через Node.js, упаковка в установщик `.exe` относится к следующей итерации;
- шифрование отдельных чувствительных полей и векторная память запланированы после стабилизации основной модели данных.

Подробный статус реализации находится в [docs/IMPLEMENTATION.md](docs/IMPLEMENTATION.md).
